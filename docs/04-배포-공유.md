# 04. 배포 & 공유 계획

## 배포 형태

GitHub 공개 저장소 + GitHub Releases에 Windows 빌드 업로드.

- **Setup exe** (NSIS 인스톨러): 일반 사용자용, 자동 시작 등록 가능
- **Portable exe**: 설치 없이 실행하고 싶은 사용자용

## electron-builder 설정 요점

```yaml
appId: com.<깃허브아이디>.claude-usage-widget
productName: Claude Usage Widget
win:
  target: [nsis, portable]
  icon: assets/icon.ico
nsis:
  oneClick: true
  deleteAppDataOnUninstall: false
publish:
  provider: github
```

## 저장소 준비물

- `README.md` (영어 기본 + 한국어 병기): 스크린샷/GIF, 설치법, 요구사항(Windows + Claude Code 로그인), 보안·프라이버시 설명, "비공식 도구" 면책 문구
- `LICENSE`: MIT
- `PRIVACY.md`: 토큰 처리 방식, 통신 대상이 api.anthropic.com뿐임을 명시
- 스크린샷: 테마 프리셋별 위젯 모습 (커스터마이징이 셀링 포인트)

## 릴리스 절차

1. `package.json` 버전 업데이트 + `CHANGELOG.md` 작성
2. 로컬 빌드 & 스모크 테스트: `npm run build` → exe 실행 확인
3. `git tag v1.0.0 && git push origin main v1.0.0`
4. GitHub Actions가 태그 push 시 자동 빌드 + Release 업로드 (`.github/workflows/release.yml`)

## 주의사항

- **코드 서명 없음** → Windows SmartScreen 경고 발생. README에 "추가 정보 → 실행" 안내 추가 (개인 오픈소스의 일반적 한계)
- 백신 오탐 가능성 → 소스 공개 + 빌드 재현 가능성으로 대응
- 비공식 API 사용 → API 변경 시 빠른 패치를 위해 파싱 로직을 한 모듈에 격리
- 이름에 "Claude" 사용 시 면책 문구 필수: "Anthropic과 무관한 커뮤니티 프로젝트"
