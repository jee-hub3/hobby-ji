# 05. Claude Code 실행 가이드

이 폴더에서 Claude Code를 열고 Phase 순서대로 진행한다.
각 Phase 끝의 **확인 기준**을 통과한 뒤 다음으로 넘어갈 것.

## 사전 준비

```
node --version   # 20+
claude --version # 로그인 상태 확인
```

## Phase 1 — 데이터 코어 (가장 먼저, UI 없이)

프롬프트:

> docs/02-아키텍처.md를 읽고, Electron 없이 Node 스크립트로 먼저 검증해줘.
> ~/.claude/.credentials.json에서 토큰을 읽어 https://api.anthropic.com/api/oauth/usage 를
> 호출하고 (헤더: Bearer 토큰, anthropic-beta: oauth-2025-04-20, User-Agent: claude-code/버전),
> **응답 JSON 전체를 콘솔에 출력**해줘. 그 응답 구조를 보고 세션(5h)/주간(7d) 사용률과
> 리셋 시각을 파싱하는 parseUsage() 함수를 작성하고, 응답 스키마를 docs/api-응답구조.md로 기록해줘.

**확인 기준**: 콘솔에 내 실제 사용률·리셋 시각이 정확히 출력됨. `docs/api-응답구조.md` 생성됨.

## Phase 2 — 위젯 MVP

프롬프트:

> docs/01-요구사항.md의 MVP(F-01~F-06)와 docs/02-아키텍처.md의 프로젝트 구조대로
> Electron 앱을 만들어줘. Phase 1의 api/파싱 코드를 src/main/에 통합하고,
> 프레임리스·투명·항상 위 위젯 창에 docs/03-UI-커스터마이징.md의 가로형 기본 디자인으로
> 세션/주간 바와 초기화 카운트다운을 표시해줘. 90초 폴링, 드래그 이동+위치 기억 포함.
> contextIsolation 유지, 토큰은 렌더러에 노출 금지.

**확인 기준**: `npm start`로 위젯이 뜨고, 실제 사용량 표시, 드래그 이동, 재시작 시 위치 유지.

## Phase 3 — 커스터마이징 + 트레이

프롬프트:

> docs/03-UI-커스터마이징.md대로 구현해줘: 그라데이션 프리셋 6종+커스텀 색상 피커,
> 레이아웃 3종(가로/세로/컴팩트), 투명도 슬라이더, 설정 패널(탭 3개), 변경 즉시 반영.
> docs/01-요구사항.md의 F-10(트레이), F-11(임계값 경고+알림)도 포함.
> 설정은 electron-store로 저장 (스키마는 03 문서 참고).

**확인 기준**: 설정에서 테마·레이아웃 클릭 시 위젯이 즉시 바뀜. 재시작 후 설정 유지. 트레이에서 숨김/종료 동작.

## Phase 4 — 빌드 & 배포

프롬프트:

> docs/04-배포-공유.md대로 electron-builder 설정(nsis+portable), 영어/한국어 README,
> LICENSE(MIT), PRIVACY.md, 그리고 태그 push 시 자동 릴리스하는
> .github/workflows/release.yml을 작성해줘. npm run build로 exe가 나오는지 확인해줘.

**확인 기준**: `dist/`에 Setup exe와 portable exe 생성, 다른 PC(또는 새 폴더)에서 실행 확인.

이후: GitHub 저장소 생성 → push → 태그 → 자동 릴리스 → 커뮤니티 공유 (04 문서 참고).

## 트러블슈팅 메모

- 401 응답 → `claude update` 실행 후 재시도 (토큰 자동 갱신)
- 투명 창이 검게 나오면 `transparent: true` + 배경 CSS `background: transparent` 확인
- SmartScreen 경고는 정상 (코드 서명 없음) — 04 문서 참고
