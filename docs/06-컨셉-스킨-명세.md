# 06. 컨셉 스킨 명세 (v0.2)

위젯에 **컨셉(스킨) 4종**을 도입한다. 사용자는 설정에서 컨셉을 선택할 수 있고, 위젯·설정 패널 모두 해당 컨셉의 시각 언어로 전환된다.

- Figma 원본: https://www.figma.com/design/rmXEPTm8ciEKjCqVlZEVsY (페이지: Widget Mockups / Concept — Aurora Glass / Concept — Neo Brutal / Concept — Editorial Ledger)
- 설정 스키마: `theme.concept: "classic" | "aurora" | "brutal" | "ledger"` (기본 `classic`, 기존 설정 파일과 하위 호환 필수)
- 공통 유지: 레이아웃 3종(가로/세로/컴팩트) × 크기 S/M/L × 투명도, 동작 설정, 임계값 경고(% 숫자 색: warn `#f0a020` / danger `#e5484d` — 컨셉별 명시 없으면 이 값)

## 구현 방식 권장

- `<body data-concept data-layout data-size>` + 컨셉별 CSS 파일 (`widget-aurora.css` 등) 또는 단일 CSS 내 `[data-concept=...]` 스코프
- DOM은 컨셉별 템플릿 분기 허용 (aurora 링, brutal 블록, ledger 눈금자는 구조가 다름). 렌더 데이터 계약은 동일: `{sessionPct, weeklyPct, sessionResetText, weeklyResetText, updatedText}`
- Aurora 링 게이지: SVG `stroke-dasharray` 또는 CSS `conic-gradient` + mask. 트랙 = 흰색 9%, 진행 아크 끝은 둥글게
- 색상 프리셋은 **컨셉별 목록**으로 분리 (`presets.js` → 컨셉별 배열). 커스텀 색상 피커는 모든 컨셉에서 유지

## 1) classic (기존) — 배경 스타일 옵션 추가

바/레이아웃은 현재 구현 그대로, 기존 그라데이션 프리셋 6종 유지.
**배경이 다크/라이트 2종 → "배경 스타일" 8종으로 확장된다** (Figma "Widget Mockups" 페이지의 시안 섹션 참조).

- 스키마: `theme.background`(현재 "dark"|"light" 문자열)를 `theme.bgStyle`로 확장. 하위 호환: 기존 `"dark"`→`flat-dark`, `"light"`→`flat-light`
- 라이트 계열(`flat-light`, `coolfog`, `warmpaper`, `holofoil`)은 텍스트를 잉크색으로 자동 반전: 텍스트 `#292420`(dim은 동일 불투명도 유지), 트랙 `rgba(0,0,0,0.10)`
- 모든 스타일은 CSS `background` 레이어(linear/radial/conic-gradient)만으로 구현 — 추가 DOM 불필요. 블러 블롭은 radial-gradient 감쇠로 근사
- 전체 불투명도(투명도 설정)는 기존 방식 유지

| id | 이름 | 값 |
|---|---|---|
| `flat-dark` | 다크 (기본) | 현행 `rgba(24,22,20,var(--opacity))` |
| `flat-light` | 라이트 | 현행 `rgba(250,248,246,var(--opacity))` |
| `coolfog` | Cool Fog | linear 135° `#F0F2F6 → #DFE4EC` — 라이트 |
| `warmpaper` | Warm Paper | linear 135° `#F6F0E7 → #EADDCE` — 라이트 |
| `meshaurora` | Mesh Aurora | base `#14121C` + radial 블롭 2개: 좌상 웜(세션 from색, 불투명도 0.20, 반경≈위젯의 45%), 우하 쿨(주간 from색, 0.16, 반경≈50%). **블롭 색은 현재 프리셋의 세션/주간 색과 연동** |
| `holofoil` | Holo Foil | conic(≈30° 시작) `#FFD9E8 → #C9E7FF → #D5FFE8 → #FFF3C4 → #FFD9E8` + 30° 사선 흰색 광택 레이어(linear, 중앙 55% → 양끝 0%) — 라이트 |
| `emerald` | Emerald Velvet | linear 135° `#0E4D3A → #03150F` + 좌상 radial `#2BE8A0` 0.20 + 우하 radial `#F5C15C` 0.22 |
| `magma` | Magma | linear 135° `#4A0F0C → #170404` + 좌하 radial `#FF4D1C` 0.45 + 우상 radial `#FFB347` 0.30 |

**볼드 배경(emerald/magma) 권장 바 색** — 기본 프리셋(주황/파랑)과 경합하므로, 해당 배경 선택 시 설정 패널에 "추천 조합 적용" 버튼 노출(강제 아님):

- emerald: 세션 `#F5C15C→#FFE29B`(골드), 주간 `#2BE8A0→#8FF5CD`(민트)
- magma: 세션 `#FFB347→#FFE08A`(앰버), 주간 `#FFE9D6→#FFB59B`(크림)

설정 UI: "테마" 섹션에 "배경 스타일" 8종 스와치 그리드 추가 (선택 = 테두리 강조, 즉시 반영).

## 2) aurora — Aurora Glass (글래스 + 듀얼 링)

**토큰**

| 항목 | 값 |
|---|---|
| 패널 배경 | `rgba(13,11,22,0.72)` + backdrop-blur 24px |
| 테두리 | 1.2px, 흰색 그라데이션 (위 35% → 아래 12% 불투명도) |
| 라운드 | 가로 30px / 세로 26px / 컴팩트 = 높이 절반(필 형태) |
| 그림자 | `0 14px 38px -8px rgba(0,0,0,0.45)` |
| 내부 글로우 | 블러(55~80px) 처리된 색 원 2개: 웜(세션 색) 좌상, 쿨(주간 색) 우하 — 프리셋 색과 연동 |
| 텍스트 | 크림 화이트 `#FFF7F0`, dim = 55% |
| 폰트 | 숫자/영문 Inter 계열(시스템: "Segoe UI"), 한글 "Malgun Gothic" |

**레이아웃**

- 가로 340×160: 좌측 듀얼 링(바깥=세션 108px 두께 9, 안쪽=주간 66px 두께 6, 중앙 21px Bold "68%" + 9px dim "세션"), 우측 정보 컬럼: "CLAUDE"(letterspacing 2.2px) + LIVE 배지(녹색 펄스 도트 + 글로우), 세션/주간 블록(그라데이션 세로 틱 3×10 + 라벨 8.5px dim → 리셋 시각 12.5px → 서브 8.5px 45%), 우하단 `↻ 14:32` 8px 35%
- 세로 150×244: 링 상단 중앙(세션 86/주간 60, 중앙 15px "68%"), 아래 "CLAUDE", 세션/주간 정보 중앙정렬 2블록
- 컴팩트 252×58 필: 미니 링 2개(34px, 두께 4.5) + 각 "68%"/"41%" 11px Bold + 우측 `↻ 2:14` 9px 50%

**프리셋 (세션 그라데이션 × 주간 그라데이션, 글로우 색 연동)**

| 이름 | 세션 | 주간 |
|---|---|---|
| Ember × Indigo (기본) | `#FF8C59→#FFC46B` | `#7C5CFF→#5CD6FF` |
| Flare × Ocean | `#FF5E7A→#FF9E6B` | `#38C6FF→#6BE0E0` |
| Lime × Violet | `#B8E986→#7FD96B` | `#9B59FF→#C89BFF` |
| Gold × Teal | `#FFC46B→#FFE29B` | `#2BD9C8→#6BE8DC` |

**설정 패널**: 다크 글래스(라운드 24, 우상단 쿨 글로우). 프리셋 = 글로우 칩(62×40, 선택 시 흰 테두리 1.6px), 슬라이더 = 웜 그라데이션 필 + 흰 원형 노브, 레이아웃 = 필 세그먼트(선택 = 웜 그라데이션 배경), 토글 = 필 스위치(on = 웜 그라데이션)

## 3) brutal — Neo Brutal (플랫 + 하드섀도우 + 블록)

**토큰**

| 항목 | 값 |
|---|---|
| 카드 배경 | `#FFFCF2` (불투명 — 투명도 설정은 전체 창 opacity로 적용) |
| 테두리 | 3px solid `#121212`, 라운드 14px (컴팩트 10px) |
| 그림자 | 하드 오프셋 `7px 7px 0 #121212` (컴팩트 4px, 내부 요소 2~3px), 블러 0 |
| 포인트 | 옐로 `#FFD433` (형광펜 밑줄, 배지, 활성 상태) |
| 기본 색 | 세션 `#FF6B36`, 주간 `#3B82F6` |
| 폰트 | 한글 최대 굵기(font-weight 900), 숫자 굵게 |

**레이아웃**

- 가로 340×180: "CLAUDE 사용량" 17px 900 + 옐로 밑줄 7px(텍스트 뒤), 우상단 회전 4° 옐로 스티커 배지 "리셋까지 2:14"(테두리 2.5px+하드섀도우 3px), 게이지 = 10칸 블록(16×20, 라운드 3, 테두리 2px, 채운 칸만 컬러) + 라벨 12px + % 17px Bold, 푸터 "세션 → 오후 4:46 · 주간 → 금 오전 9:00" 10.5px 65%
- 세로 190×250 배터리 타워: 블록 28×13 세로 10칸, **아래에서부터 차오름**, 하단에 % 14px 900 + 라벨 10px
- 컴팩트 310×52: 라벨 + 미니 블록 5칸(12×16, 1칸=20%) + % 13px 900 ×2

**프리셋 (플랫 페어)**: TOMATO×SKY `#FF6B36/#3B82F6`(기본), LEMON×GRAPE `#FFD432/#8B5CF6`, MINT×CORAL `#2DD4A8/#FF5E7A`, INK×PAPER `#1F1F1F/#B8B2A6`

**설정 패널**: 크림 카드 + 3px 테두리 + 하드섀도우 7px. 닫기 = 옐로 사각 버튼, 프리셋 = 스와치 2개 든 카드(선택 = 옐로 배경+하드섀도우), 슬라이더 = 흰 트랙 2.5px 테두리 + 옐로 필 + 사각 노브, 레이아웃 = 버튼 3개(선택 = 옐로+하드섀도우), 체크박스 = 22px 사각(체크 시 옐로+✓)

## 4) ledger — Editorial Ledger (신문 조판)

**토큰**

| 항목 | 값 |
|---|---|
| 종이 배경 | `#F8F5EF` (불투명) |
| 잉크 | `#1A1A1A`, dim = 50%, 헤어라인 = 15~25% |
| 악센트 | 단일색, 기본 Terracotta `#C15F3C` — %기호·LIVE 도트·눈금 마커에만 사용 |
| 라운드 | 6px (컴팩트 4px), 그림자 은은한 `0 10px 24px -6px rgba(0,0,0,0.3)` |
| 조판 | 마스트헤드 "CLAUDE USAGE" letterspacing 2px + 더블 룰(2px+1px), 숫자 대형 Bold |

**레이아웃**

- 가로 416×227: 마스트헤드+LIVE 도트 → 더블 룰 → 세로 헤어라인 디바이더로 나눈 2단(세션|주간): 라벨 Bold+한도 설명 dim → 46px 숫자+테라코타 % → 눈금자(0/25/50/75/100 틱 + 잉크 사용선 2px + 테라코타 세로 마커) → 리셋 텍스트 → 헤어라인 푸터("마지막 갱신 14:32" | "2026.07.11 FRI")
- 세로 180×262: 1단 칼럼, 마스트헤드 9px → 더블 룰 → 세션(34px 숫자) → 눈금자 → 주간 → 푸터
- 컴팩트 276×46 티커: 상하 1.5px 룰, "세션 **68**% · 2:14 후 │ 주간 **41**% · 금 09:00" (숫자 17px, 세로 디바이더)

**프리셋 (단일 악센트 잉크)**: Terracotta `#C15F3C`(기본), Prussian `#2C4770`, Forest `#3E5F44`, Burgundy `#7A2E3A`

**설정 패널**: 종이 배경, "SETTINGS 설정" 마스트헤드 + 더블 룰. 잉크 선택 = 색 도트 18px(선택 = 잉크색 외곽 링), 슬라이더 = 헤어라인+틱+테라코타 마커, 레이아웃 = 텍스트 선택(활성 = Bold + 테라코타 2px 밑줄), 동작 = 라디오(선택 = 테라코타 도트)

## 설정 UI 공통 변경

1. 설정 패널 최상단에 **"컨셉" 섹션 신설**: 4개 카드(classic/aurora/brutal/ledger) — 각 컨셉의 미니 프리뷰(CSS로 그린 썸네일)와 이름, 클릭 즉시 위젯+설정 패널 전환
2. "테마" 섹션의 프리셋 목록은 **현재 컨셉의 프리셋으로 교체** 표시
3. 컨셉 전환 시 레이아웃/크기/투명도/동작 설정은 유지

## 수용 기준 (QA)

- [ ] 4개 컨셉 × 3레이아웃 × 3크기 = 36조합 모두 잘림/겹침 없이 렌더 (창 크기는 콘텐츠 측정 방식 유지)
- [ ] classic 배경 스타일 8종 모두 3레이아웃에서 렌더 확인, 라이트 4종에서 텍스트 잉크 반전 정상
- [ ] 기존 설정의 background:"dark"/"light"가 flat-dark/flat-light로 자동 이행
- [ ] 컨셉 전환·프리셋 변경·투명도 즉시 반영 (재시작 불필요), 재시작 후 유지
- [ ] 기존 사용자 설정 파일(concept 키 없음) 로드 시 classic으로 동작, 기존 프리셋 유지
- [ ] 오류/온보딩 화면(로그인 필요 등)도 4컨셉 모두에서 스타일 깨짐 없음
- [ ] 임계값 경고(80/95%)가 4컨셉 모두에서 시각적으로 표현됨
