# 02. 기술 아키텍처

## 스택

- **Electron** (최신 LTS) + 순수 HTML/CSS/JS (프레임워크 없이 시작, 필요 시 도입)
- **electron-builder**: Windows NSIS 인스톨러 + portable exe 빌드
- **electron-store**: 설정 저장
- Node 20+

## 데이터 소스 (검증 완료)

기존 오픈소스 위젯들이 공통으로 쓰는 방식. Anthropic 비공식 API.

### 1) 토큰 읽기

```
경로: %USERPROFILE%\.claude\.credentials.json
     (CLAUDE_CONFIG_DIR 환경변수가 있으면 그 경로 사용)
필드: claudeAiOauth.accessToken
```

### 2) 사용량 조회

```
GET https://api.anthropic.com/api/oauth/usage
Headers:
  Authorization: Bearer <accessToken>
  Content-Type: application/json
  anthropic-beta: oauth-2025-04-20
  User-Agent: claude-code/<설치된 버전>   ← `claude --version`으로 획득
```

응답에 세션(5h)·주간(7d) 및 모델별 한도의 사용률과 리셋 시각(ISO)이 포함됨.
**⚠️ 스키마는 비공식이므로 Phase 1에서 실제 응답을 덤프해 파싱 코드를 작성할 것.**

### 3) 계정 정보 (선택)

```
GET https://api.anthropic.com/api/oauth/profile  (동일 헤더)
```

### 오류 처리 규칙

- `401`: 토큰 만료 → 백그라운드에서 `claude update` 실행해 토큰 갱신 시도(기존 도구 검증된 기법), 실패 시 "Claude Code에 다시 로그인하세요" 표시
- `429`: Retry-After 헤더만큼 폴링 백오프
- 네트워크 오류: 마지막 데이터 유지 + "오래된 데이터" 뱃지

## 프로세스 구조

```
Main Process (Node)
 ├─ credentials.js   토큰 읽기 (유일하게 토큰을 다루는 모듈)
 ├─ api.js           usage/profile 호출, 오류 처리 (유일한 네트워크 모듈)
 ├─ poller.js        폴링 스케줄 (기본 90초, 유휴/잠금 시 정지, 리셋 직전 가속)
 ├─ tray.js          트레이 아이콘/메뉴
 ├─ settings.js      electron-store 래퍼
 └─ windows.js       위젯 창 + 설정 창 생성
      │  IPC (contextBridge, preload.js — nodeIntegration 금지)
      ▼
Renderer
 ├─ widget/   위젯 UI (index.html, widget.css, widget.js)
 └─ settings/ 설정 패널 UI
```

## 위젯 창 옵션

```js
new BrowserWindow({
  frame: false, transparent: true, alwaysOnTop: true,
  skipTaskbar: true, resizable: false,
  webPreferences: { preload, contextIsolation: true }
})
```

드래그 이동: CSS `-webkit-app-region: drag`. 위치는 settings에 저장.

## 프로젝트 폴더 구조

## 보안 원칙 (공유 시 신뢰의 핵심)

1. 네트워크 통신은 `api.anthropic.com`만 — api.js 한 파일에 격리
2. 토큰은 메모리에서만 사용, 로그·설정파일·렌더러에 절대 노출 금지
3. eval/동적 코드 실행 금지, 의존성 최소화
4. README에 보안 구조 명시 (감사 가능하게)
